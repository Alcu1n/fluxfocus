## 1. 产品概述

FluxFocus 是一款效率与专注类 iOS 产品，核心交互为 `iPhone 触碰桌面 NFC 摆件（NTAG216）` 后进入专注流程。  
产品目标是将“神圣座位、链式时延、工作量证明、下必为例、RSIP”落地为可执行的行为系统，而不是普通计时器或打卡工具。

本版本 PRD 明确：   
- 专注系统完全由规则引擎、状态机、数据统计与可解释策略实现。  
- App Clip 作为 NFC 触发入口。  
- “家长控制模式”（Focus Shield）作为专注期 App 访问限制。

## 2. 产品定位与边界

### 2.1 定位

- 面向重度拖延、启动困难、专注中断频繁用户。  
- 非医疗产品，不提供诊断或治疗声明。  
- 通过硬件仪式入口与行为规则提高执行力。

### 2.2 价值主张

- 快速启动：触碰摆件即可进入专注。  
- 强约束：会话中可限制特定 App 打开。  
- 可累积：工作量证明链持续增长。  
- 可演化：通过判例与定式树持续优化策略。

### 2.3 非目标

- 不做社交网络、排行榜。  
- 不做内容社区。  
- 不做泛 GPT 助手。  
- 不做后台无感自动专注识别。

## 3. 核心方法论到功能映射

- 神圣座位原理 -> “唯一 NFC 摆件 + 触碰即进入承诺场景”。  
- 链式时延原理（CTDP） -> “预约链（延时启动）+ 主链（专注执行）”。  
- 工作量证明 -> 每次完成会话生成链节点，形成沉没成本和连续性。  
- 下必为例 -> 每次疑似违规必须做规则判决：断链重置或永久放宽。  
- RSIP -> 针对失败路径回溯到干预节点，沉淀“定式树”。

## 4. 用户画像与使用场景

### 4.1 目标用户

- 学生：备考、自习、论文写作。  
- 知识工作者：深度工作、写作、研发。  
- 自控困难用户：易分心、启动困难、习惯性拖延。

### 4.2 核心场景

- 下班回家后无法开始任务，触碰摆件进入专注。  
- 专注时想打开短视频/社交 App，被系统拦截。  
- 会话失败后进行判决并重置或放宽规则。  
- 周期性复盘时查看链条稳定度并调整策略。

## 5. 版本规划总览

- 阶段一：MVP（8-12 周）  
目标：验证 `NFC -> App Clip -> 专注会话 -> 链条` 闭环可用、可持续。  

- 阶段二：后续阶段（12-24 周）  
目标：扩展 RSIP、强化多场景与多摆件策略，形成完整行为系统。

## 6. 阶段一（MVP）PRD

### 6.1 MVP 目标

- 完成核心链路并达到可发布内测标准。  
- 完成 Focus Shield 基础能力。  
- 在纯本地数据架构下稳定运行。

### 6.2 MVP 范围（In Scope）

- NFC 摆件绑定与识别。  
- App Clip 触发与品牌化入口 Sheet。  
- 主链会话：开始、进行、完成、失败。  
- 预约链：延时启动和履约判定。  
- 工作量证明链：节点累计与断链。  
- 下必为例规则判决。  
- 家长控制模式：会话期 App 屏蔽。  
- 本地统计面板：日/周会话、完成率、断链次数。

### 6.3 MVP 非范围（Out of Scope）

- 云同步与多端实时一致性。  
- 团队协作、社交功能。  
- 智能推荐与自动生成策略。  
- 可穿戴设备联动（首期不做 Apple Watch）。

### 6.4 MVP 关键用户旅程

1. 触碰 NFC 摆件。  
2. 系统展示 App Clip 卡片并进入入口 Sheet。  
3. 用户选择时长、目标、是否开启 Focus Shield。  
4. 开始专注会话。  
5. 会话中触发违规事件时，进入“下必为例”判决。  
6. 会话结束记录 PoW 节点或断链事件。  
7. 用户查看统计与链条状态。

### 6.5 MVP 功能需求详细

#### 6.5.1 NFC 与 App Clip

- 标签采用 NTAG216，写入 NDEF URL。  
- URL 绑定 App Clip Experience。  
- 触碰后 2 秒内应出现 App Clip 卡片。  
- App Clip 首屏为“进入专注模式”精美 Sheet，包含：  
  - 今日链条状态  
  - 会话时长选择  
  - Focus Shield 开关  
  - 开始按钮  
- 支持从 App Clip 跳转完整 App 并保持上下文。

#### 6.5.2 专注会话引擎（主链）

- 会话状态机：`Idle -> Ready -> Running -> Completed/Failed`。  
- 运行中限制非必要导航，突出剩余时长与目标。  
- 结束条件：  
  - 到达设定时长且无致命违规 -> Completed  
  - 用户放弃或违规判定断链 -> Failed  
- Completed 生成主链节点，Failed 记录断链事件。

#### 6.5.3 预约链（链式时延）

- 默认支持 `15 分钟后开始`。  
- 预约后必须在窗口期内触发主链。  
- 未履约触发预约失败判决。  
- 预约链与主链分别维护节点计数。

#### 6.5.4 下必为例判例引擎

- 违规类型（MVP）：  
  - 会话中主动退出  
  - 长时离开前台  
  - 尝试打开被屏蔽 App  
  - 人工标记“状态不合格”  
- 每次违规必须二选一：  
  - 断链重置  
  - 永久允许该类行为（作用于当前协议版本）  
- 所有判例可追溯查看。

#### 6.5.5 家长控制模式（Focus Shield）

- 使用 iOS Screen Time 相关框架完成 App 屏蔽。  
- 用户在会话前选择禁用 App 列表。  
- 会话开始自动启用，结束自动解除。  
- 禁用 App 被打开时显示拦截提示。  
- 拦截事件写入违规日志并进入判例流程。

#### 6.5.6 数据统计与反馈

- 今日专注时长。  
- 本周完成会话数。  
- 主链/预约链长度。  
- 断链次数与断链原因分布。  
- Focus Shield 拦截次数。

### 6.6 MVP 信息架构

- 首页：今日状态、触碰指引、快速开始。  
- 会话页：倒计时、目标、退出判决入口。  
- 链条页：主链与预约链历史。  
- 判例页：违规事件与生效规则。  
- 设置页：NFC 标签管理、Focus Shield 配置、数据导出。

### 6.7 MVP 技术架构

#### 6.7.1 客户端架构

- 平台：iOS（SwiftUI）。  
- 能力：CoreNFC、App Clip、FamilyControls、ManagedSettings、DeviceActivity、UserNotifications。  
- 模块划分：  
  - `NFCModule`：标签识别、签名校验、配对  
  - `SessionModule`：会话状态机  
  - `ChainModule`：主链/预约链 PoW 记录  
  - `PrecedentModule`：违规判例引擎  
  - `ShieldModule`：家长控制模式  
  - `StatsModule`：本地统计

#### 6.7.2 数据层

- 纯本地存储（SwiftData 或 SQLite）。  
- 不引入云端主档。  
- 提供本地导出/导入（加密文件）能力。

#### 6.7.3 NFC 规范

- 标签 URL 结构：`https://clip.<domain>/i/{tagPublicId}?v=1&s={sig}`  
- `tagPublicId` 用于业务标识。  
- `sig` 用于防伪造校验。  
- 生产阶段建议标签锁定为只读。

### 6.8 MVP 数据模型

- `Tag`: id, tagPublicId, uidHash, name, status, createdAt  
- `FocusSession`: id, tagId, goal, durationSec, startAt, endAt, status, source  
- `ChainNode`: id, chainType(main/appointment), index, sessionId, proofHash, createdAt  
- `ViolationEvent`: id, sessionId, type, payload, createdAt  
- `PrecedentRule`: id, violationType, decision(reset/allowForever), scope, createdAt  
- `ShieldPolicy`: id, selectionToken, enabled, updatedAt  
- `DailyMetrics`: date, focusSec, completedCount, failCount, shieldBlocks

### 6.9 MVP 验收标准

- NFC 触碰到 App Clip 弹出成功率 >= 95%。  
- 入口 Sheet 到会话启动转化率 >= 90%。  
- Focus Shield 会话期屏蔽生效率 >= 99%。  
- 所有违规事件必须进入可判决流程，漏判率为 0。  
- 会话记录与链条记录一致性错误率 < 0.5%。

### 6.10 MVP 测试计划

- 功能测试：NFC、Clip、会话、链条、判例、Shield。  
- 异常测试：无网、权限拒绝、标签损坏、强杀恢复。  
- 兼容测试：不同机型、不同 iOS 次版本。  
- 回归测试：协议更新不破坏历史数据。

## 7. 阶段二（后续阶段）PRD

### 7.1 阶段目标

- 从“单会话约束”升级到“长期稳态迁移系统”。  
- 完整实现 RSIP 定式树。  
- 支持多摆件多场景，强化产品长期留存。

### 7.2 功能范围（In Scope）

- RSIP 定式树管理。  
- 多协议模板系统。  
- 多摆件体系。  
- 高级 Focus Shield 策略。  
- 高级统计分析。  
- 商业化订阅分层。

### 7.3 核心功能详细

#### 7.3.1 RSIP 定式树

- 支持手动新增“定式节点”：  
  - 触发条件  
  - 干预动作  
  - 成功判据  
  - 失败回滚  
- 节点关系支持父子依赖。  
- 失败时允许子树剪枝。  
- 提供“根国策”约束，保证系统持续运行。

#### 7.3.2 协议模板系统

- 模板类型：学习、写作、运动、睡前、出门。  
- 每个模板定义：  
  - 主链时长  
  - 预约链窗口  
  - 违规策略  
  - Shield 策略  
- 支持用户基于模板创建个人协议版本。

#### 7.3.3 多摆件体系

- 支持多个 NFC 摆件绑定不同场景。  
- 摆件与协议一对一或一对多绑定。  
- 处理标签更换、丢失、作废与迁移。  
- 增加批量写码与验签工具流程（运营/工厂侧）。

#### 7.3.4 高级 Focus Shield

- 模式分级：轻专注、深专注、极限专注。  
- 支持按时段策略应用屏蔽规则。  
- 支持会话内临时白名单（需记录审计）。  
- 屏蔽尝试可映射为不同严重度违规。

#### 7.3.5 高级统计与复盘

- 周/月趋势图。  
- 失败路径分布。  
- 定式命中率。  
- 稳定度评分（基于完成率、断链率、履约率）。  
- 规则变更影响分析（变更前后对比）。

### 7.4 后续阶段技术架构增强

- 引入规则 DSL，避免硬编码策略。  
- 引入迁移框架支持协议版本升级。  
- 引入可选匿名遥测（默认关闭）用于产品优化。  
- 加强崩溃恢复与一致性校验任务。

### 7.5 后续阶段数据模型新增

- `ProtocolTemplate`: id, name, config, createdAt  
- `ProtocolVersion`: id, templateId, ruleset, effectiveFrom, deprecatedAt  
- `PolicyTree`: id, name, rootNodeId, status  
- `PolicyNode`: id, treeId, parentId, trigger, action, rollback, score  
- `TagBinding`: id, tagId, protocolVersionId, scene, active

### 7.6 后续阶段验收标准

- 使用定式树后 4 周断链率下降 >= 20%。  
- 多摆件切换成功率 >= 99%。  
- 协议升级后历史数据可读率 100%。  
- 高级 Shield 策略配置错误率 < 1%。

### 7.7 后续阶段测试计划

- 定式树结构一致性测试。  
- 多摆件并发与冲突测试。  
- 大数据量统计性能测试。  
- 版本迁移与回滚测试。  
- 订阅权益边界测试。

## 8. 商业化设计（无 AI）

### 8.1 收费模型

- 硬件首购：NFC 摆件。  
- 软件订阅：高级模板、定式树深度功能、高级统计、导出能力。

### 8.2 订阅分层示例

- Free：基础会话、基础链条、基础 Shield。  
- Pro：多协议模板、定式树、月报、数据导出。  
- Pro+：多摆件高级管理、企业/家庭套件（后续）。

## 9. 指标体系

- 激活：NFC 首次触碰后 24h 内完成首会话比例。  
- 转化：App Clip 到完整 App 安装率。  
- 行为：周有效会话数、完成率、预约履约率。  
- 约束：Shield 拦截频次、违规事件密度。  
- 留存：7 日、30 日留存。  
- 商业：付费转化率、续费率、硬件复购率。

## 10. 风险与对策

- iOS 权限与审核风险：提前准备 Screen Time/FamilyControls 合规说明与演示视频。  
- NFC 物理稳定性风险：制定标签质检和容错重绑流程。  
- 规则过严导致流失：提供协议模板和渐进式强度。  
- 纯本地数据丢失风险：默认提醒用户启用加密备份导出。

## 11. 里程碑建议

- M1（第 2 周）：NFC + App Clip 触发链路跑通。  
- M2（第 4 周）：会话引擎 + 链条 + 判例引擎完成。  
- M3（第 6 周）：Focus Shield 完成并稳定。  
- M4（第 8 周）：MVP 内测。  
- M5（第 12 周）：MVP 公测。  
- M6（第 20 周）：RSIP 与多摆件能力上线。
