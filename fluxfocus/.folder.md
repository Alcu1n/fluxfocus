# Folder: fluxfocus

> Trigger: When this folder's structure/responsibilities/file list changes, update this document.
> 触发条件:当本文件夹的结构/职责/文件列表变化时,更新此文档。

This folder owns the full app UI, local state, NFC read/write, and App Clip handoff acceptance. / 本文件夹负责完整 App UI、本地状态、NFC 读写与 App Clip 移交接收。
It consumes hosted invocation URLs and custom scheme handoffs, then produces persisted sessions, chains, and settings. / 它消费托管 invocation URL 与自定义 scheme handoff，并产出持久化会话、链条与设置数据。
The invariant is that every physical tag maps to one short `/i/<tagPublicId>` URL and one matching `fluxfocus://focus/<tagPublicId>` handoff. / 最重要的不变量是每张物理标签都对应一个短 `/i/<tagPublicId>` URL 与一个匹配的 `fluxfocus://focus/<tagPublicId>` handoff。

## Files
- `ContentView.swift`: composition root - tabs, invocation handling, and settings UI / 标签页、invocation 处理与设置界面
- `FluxFocusAppStore.swift`: domain service - URL generation, parsing, and state transitions / URL 生成、解析与状态流转
- `FluxFocusModels.swift`: schema - SwiftData entities and supporting value types / SwiftData 实体与支撑值类型
- `Info.plist`: bundle config - full app metadata and custom handoff URL scheme / 完整 App 元数据与自定义 handoff URL scheme
- `NFCManager.swift`: device service - CoreNFC read/write bridge / CoreNFC 读写桥接
- `fluxfocusApp.swift`: app entry - model container and environment wiring / 应用入口、模型容器与环境注入
- `fluxfocus.entitlements`: capabilities - NFC and associated domains entitlements / NFC 与关联域权限配置
- `Assets.xcassets/`: assets - app icon and colors / 图标与颜色资源
