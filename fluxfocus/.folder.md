# Folder: fluxfocus

> Trigger: When this folder's structure/responsibilities/file list changes, update this document.
> 触发条件:当本文件夹的结构/职责/文件列表变化时,更新此文档。

This folder owns the full app UI, local state, NFC read/write, App Clip handoff acceptance, Family Controls shielding, and the wheel-based session studio. / 本文件夹负责完整 App UI、本地状态、NFC 读写、App Clip 移交接收、Family Controls 屏蔽与基于滚轮的会话控制台。
It consumes hosted invocation URLs, custom scheme handoffs, Screen Time authorization state, and WheelPickerKit UI primitives, then produces persisted sessions, chains, settings, and live shields. / 它消费托管 invocation URL、自定义 scheme handoff、屏幕使用时间授权状态与 WheelPickerKit UI 原语，并产出持久化会话、链条、设置与实时屏蔽。
The invariant is that every physical tag maps to one short `/i/<tagPublicId>` URL, one matching `fluxfocus://focus/<tagPublicId>` handoff, one wheel-based duration draft, and at most one active shield policy. / 最重要的不变量是每张物理标签都对应一个短 `/i/<tagPublicId>` URL、一个匹配的 `fluxfocus://focus/<tagPublicId>` handoff、一份基于滚轮的时长草稿，并且最多只有一个生效的屏蔽策略。

## Files
- `ContentView.swift`: composition root - tabs, invocation handling, wheel-based session studio, and settings UI / 标签页、invocation 处理、基于滚轮的会话控制台与设置界面
- `FluxFocusAppStore.swift`: domain service - URL generation, parsing, and state transitions / URL 生成、解析与状态流转
- `FluxFocusModels.swift`: schema - SwiftData entities and supporting value types / SwiftData 实体与支撑值类型
- `FocusShieldController.swift`: shield bridge - Family Controls authorization, picker state, and ManagedSettings application / Family Controls 授权、选择器状态与 ManagedSettings 应用
- `Info.plist`: bundle config - full app metadata and custom handoff URL scheme / 完整 App 元数据与自定义 handoff URL scheme
- `NFCManager.swift`: device service - CoreNFC read/write bridge / CoreNFC 读写桥接
- `fluxfocusApp.swift`: app entry - model container and environment wiring / 应用入口、模型容器与环境注入
- `fluxfocus.entitlements`: capabilities - NFC, associated domains, and Family Controls entitlements / NFC、关联域与 Family Controls 权限配置
- `Assets.xcassets/`: assets - app icon and colors / 图标与颜色资源
