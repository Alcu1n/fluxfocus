# Folder: fluxfocusClip

> Trigger: When this folder's structure/responsibilities/file list changes, update this document.
> 触发条件:当本文件夹的结构/职责/文件列表变化时,更新此文档。

This folder owns the App Clip target, landing UI, invocation diagnostics, and handoff initiation. / 本文件夹负责 App Clip target、落地 UI、invocation 诊断与移交发起。
It receives the same short invocation URLs as the full app and can hand off to the installed app through a custom scheme. / 它接收与完整 App 相同的短 invocation URL，并可通过自定义 scheme 移交给已安装的完整 App。
The invariant is that Clip parsing accepts `/i/<tagPublicId>` and hands off using `fluxfocus://focus/<tagPublicId>`. / 最重要的不变量是 Clip 解析 `/i/<tagPublicId>`，并使用 `fluxfocus://focus/<tagPublicId>` 完成移交。

## Files
- `ClipContentView.swift`: clip surface - invocation status, local timer, and handoff actions / invocation 状态、本地计时与移交操作
- `FluxFocusClipApp.swift`: clip entry - scene and URL plumbing / Clip 入口与 URL 管线
- `Info.plist`: bundle config - App Clip metadata / App Clip 包元数据
- `fluxfocusClip.entitlements`: capabilities - associated domains entitlements / 关联域权限
- `Assets.xcassets/`: assets - clip icon catalog / Clip 图标资源
